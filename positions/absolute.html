CSS Positioning: Absolute

The container below demonstrates position: absolute. Notice how the behavior changes based on the parent's positioning context.

Parent: I am a position: relative container. My child uses position: absolute, so its position is relative to me.

Child: I am position: absolute. My top: 10px and left: 10px are relative to my parent.

Child with no positioned parent: The box below is position: absolute with no relative parent, so it is positioned relative to the viewport.

I am positioned relative to the viewport because no parent has position: relative.

Key Points:

  • position: absolute removes the element from the document flow. Other elements act as if it does not exist.
  • If no parent has a positioning context (position: relative/absolute/fixed/sticky), it positions relative to the viewport.
  • If a parent has position: relative, the absolute element positions relative to that parent.
  • Absolute elements are not block-level by default; they shrink-wrap their content.

Tip: Use position: absolute for precise positioning inside a relatively positioned container.
