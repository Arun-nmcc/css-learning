Negative Values in Box Model Properties

This example demonstrates how negative values affect different box model properties:

This is an adjacent element.
Valid Negative Margin: This element moves upward into the space of the adjacent element above it due to the negative margin.
Invalid Negative Padding: The browser ignores negative padding, so it is treated as 0.
Invalid Negative Border: The browser ignores negative border values, so it is treated as 0.
Invalid Negative Width: The browser ignores negative width values, so it is treated as 0.

Key Points:

  • Negative Margin: Valid. Moves the element closer to (or overlaps with) its adjacent elements.
  • Negative Padding: Invalid. Padding cannot have negative values; browsers treat it as 0.
  • Negative Border: Invalid. Border width cannot be negative; browsers treat it as 0.
  • Negative Width/Height: Invalid. These properties cannot have negative values; browsers treat them as 0.

What Happens with Negative Margins?

A negative margin reduces the space between the element and its adjacent elements. For example:

  • Negative margin-top: Pulls the element upward, possibly overlapping the element above.
  • Negative margin-right: Pulls the element to the left, potentially overlapping the element to its right.
  • Negative margin-bottom: Pulls the element upward, reducing space below.
  • Negative margin-left: Pulls the element to the left, potentially overlapping the element to its left.
